# Dragonflight-Fix v2.0.0 全面回顾与反思

> 日期：2026-04-07
> 对比对象：Fix v2.0.0 vs _dev/ 下 DragonflightUI / Reforged / -Dragonflight3 / DragonUI 四份参考包

## Context

Dragonflight-Fix 经过多轮迭代，功能从 v1.3.3 的 38 文件/16K 行增长到 v2.0.0 的 51 文件/25K 行（+51%）。期间重心放在修bug和加新功能（天赋规划、血球、导入导出、面板美化等），但整体 UI 风格进度落后。

本文对比 fix 与 `_dev/` 下四份参考包，从 **功能** 和 **UI 风格** 两个维度做系统性回顾。

## 五、关键反思

### 做得好的地方
1. **功能创新远超预期** — 天赋规划、血球、导入导出都是独有功能，没有任何参考包有这些
2. **面板工厂模式** — paperdoll.lua 的 CreatePaperDollFrame 和 CreateRedButton 是加速面板开发的利器
3. **深度优于广度** — 角色面板 551 行（D3 仅 222 行）、拾取 1,122 行（D3 仅 91 行），做就做透
4. **13 个面板已完成** — 超过 D3 的 60%

### 需要调整的地方
1. **系统级视觉问题优先于新面板** — 滚动条换肤一次解决所有面板的一致性，比新增任何单个面板都有价值
2. **控制深度冲动** — 角色面板 551 行 vs D3 的 222 行，投入过多。未来新面板应先达到 D3 同等水平即可
3. **功能 vs 风格的平衡** — 过去倾向于先做功能（有趣），UI 风格（枯燥但影响大）需要更多时间
4. **参考 DragonUI 的暗色主题** — Fix 目前只有简单明/暗切换，DragonUI 有 3 种暗色预设 + 1,132 行实现，这是用户体验的大差距
