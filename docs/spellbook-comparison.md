## 四、技术实现差异

### D3 实现方式
- `KillFrame(SpellBookFrame)` 彻底杀死原生法术书
- 用 `CreatePaperDollFrame` 创建金属边框窗口
- 36 个按钮固定网格布局（3x6 x2页）
- 法术数据一次性遍历 `GetSpellTabInfo` + `GetSpellName`
- 简单的被动/种族/等级文字标记
- 翻页：超过 36 个法术时分页

### ModernSpellBook 实现方式
- 隐藏原生 SpellBookFrame 的所有子元素，但保留框架用于 API 兼容
- 自定义 OOP 类系统（CIcon → CSpellBookIcon → CSpellItem）
- 列表式布局，按分类分组，动态创建/复用 UI 元素
- 法术数据管道：采集 → 过滤 → 关键词构建 → 排序
- 训练师数据服务：监听 TRAINER_SHOW 事件采集未学技能
- 对象池复用机制（GetOrCreateSpellItem）
- 事件去抖（0.3s 延迟刷新）

## 五、优势分析

### D3 优势
1. **极简轻量** — 535 行 vs 4000+，加载和内存占用极小
2. **与 DF 金属边框风格统一** — 使用 CreatePaperDollFrame，与其他面板视觉一致
3. **移植到 DFUI 工作量小** — 已有 API 翻译经验，1-2 小时可完成
4. **功能够用** — 被动过滤、等级过滤、宠物 Tab、翻页、冷却、拖拽全有

### ModernSpellBook 优势
1. **功能完整度碾压** — 搜索、未学技能、高亮系统、设置菜单、法术计数
2. **独立插件** — 零依赖，可直接安装使用，不影响其他插件
3. **天赋树集成** — 完整的 3 专精天赋面板（可选）
4. **视觉精细度更高** — 圆形被动图标、发光徽章、分类标题、文字样式
5. **多语言支持** — 6 种语言
6. **用户体验** — 窗口拖拽/缩放记忆、设置菜单、Shift+点击聊天

## 六、可选方案

### 方案 A：直接安装 ModernSpellBook
- 作为独立插件放入 AddOns/
- 优点：零工作量，功能最全
- 缺点：视觉风格与 DFUI 金属边框不统一

### 方案 B：移植 D3 法术书到 DFUI
- 参考 `_dev/-Dragonflight3/mods/panels/spellbook.lua`
- 优点：与其他面板风格统一，代码量小
- 缺点：功能较基础（无搜索、无未学技能）

### 方案 C：基于 ModernSpellBook 功能，用 DFUI 框架重写
- 取 ModernSpellBook 的功能设计（搜索、未学技能、高亮）
- 用 DFUI 的 CreatePaperDollFrame 做外框
- 优点：功能+风格兼得
- 缺点：工作量最大（估计 1000-1500 行）

### 方案 D：安装 ModernSpellBook + 外观适配
- 安装 ModernSpellBook 作为独立插件
- 写一个小型适配模块，将其外框替换为 DFUI 金属边框
- 优点：功能完整 + 风格基本统一
- 缺点：适配可能有兼容性问题
